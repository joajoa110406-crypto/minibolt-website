# MiniBolt 재구매 유도 시스템 마스터플랜

## 5개 에이전트 분석 결과 종합

### 현재 구현 상태
- ✅ 주문 확인 이메일 (고객+관리자)
- ✅ 배송 상태 변경 알림 이메일
- ✅ B2B 등급별 할인 (bronze~vip)
- ✅ 벌크 할인 (5000개 묶음)
- ✅ 고객 등급 자동 산정

### 미구현 (이번 구현 대상)
- ❌ 원클릭 재주문
- ❌ 장바구니 이탈 복구 배너
- ❌ 최근 본 상품 추적
- ❌ 재구매 유도 이메일 시퀀스
- ❌ 주문 완료 후 관련 상품 추천
- ❌ 자주 구매하는 제품 대시보드
- ❌ 재주문 주기 예측 시스템

## 구현 계획 (8개 시스템)

### 1. 원클릭 재주문 버튼 [난이도: 하 / 효과: 상]
**위치**: `/orders` 페이지
**기능**: 이전 주문 내역에서 "다시 주문하기" 클릭 시 동일 상품+수량이 장바구니에 복원
**구현**:
- 주문 상세에 재주문 버튼 추가
- cart.ts에 `reorderFromHistory()` 함수 추가
- 장바구니 페이지로 자동 이동

### 2. 장바구니 이탈 복구 배너 [난이도: 하 / 효과: 중]
**위치**: 메인 페이지, 제품 목록 페이지 상단
**기능**: 장바구니에 상품이 있는 상태에서 이탈 후 재방문 시 "장바구니에 N개 상품이 있습니다" 배너 표시
**구현**:
- `CartRecoveryBanner` 컴포넌트
- localStorage 장바구니 감지
- 마지막 방문 시간 기록으로 이탈 판단 (30분 이상)

### 3. 최근 본 상품 추적 및 표시 [난이도: 하 / 효과: 중]
**위치**: 제품 목록 페이지 상단, 메인 페이지
**기능**: 최근 조회한 상품 최대 10개를 localStorage에 저장하고 캐러셀로 표시
**구현**:
- `src/lib/recently-viewed.ts` - 조회 기록 관리
- `RecentlyViewed` 컴포넌트 - 가로 스크롤 목록
- 제품 상세/제품 목록에서 조회 시 기록

### 4. 재구매 유도 이메일 시퀀스 [난이도: 중 / 효과: 상]
**기능**: 주문 완료 후 단계별 자동 이메일 발송
**시퀀스**:
- 배송 완료 +3일: "사용에 문제없으신가요?" + 재주문 링크
- 배송 완료 +14일: 관련 제품 추천 + 재주문 안내
- 마지막 주문 후 30일: "재고가 떨어지셨나요?" 리마인더
- 마지막 주문 후 60일: 휴면 고객 복구 이메일
**구현**:
- `email_sequences` Supabase 테이블
- mailer-templates.ts에 시퀀스 이메일 4종 추가
- Cron job에서 매일 발송 대상 체크

### 5. 주문 완료 페이지 관련 상품 추천 [난이도: 하 / 효과: 중]
**위치**: `/checkout/success` 페이지 하단
**기능**: 방금 구매한 상품과 같은 카테고리/규격의 다른 제품 추천
**구현**:
- 구매 상품의 category + diameter 기반 유사 상품 필터링
- "이런 제품도 함께 많이 주문합니다" 섹션

### 6. 자주 구매하는 제품 섹션 [난이도: 중 / 효과: 상]
**위치**: 메인 페이지 (로그인 사용자)
**기능**: 이전 구매 이력 기반으로 자주 주문한 상품 Top 5 표시
**구현**:
- Supabase 쿼리: 고객별 상품 구매 빈도 집계
- `FrequentProducts` 컴포넌트
- 로그인 시에만 표시, 비로그인 시 인기 상품 대체

### 7. 재주문 주기 예측 Cron [난이도: 중 / 효과: 상]
**기능**: 고객별 구매 간격을 분석하여 다음 주문 예상일 계산
**로직**:
- 고객-제품별 구매 이력에서 평균 주문 간격 계산
- 예상 재주문 시점 5일 전: 리마인더 이메일 자동 발송
- "지난번 주문하신 [제품명] 재주문 시점이 다가왔습니다"
**구현**:
- `reorder_predictions` Supabase 테이블
- Cron: `/api/cron/reorder-reminder`

### 8. Supabase 스키마 확장
**새 테이블**:
```sql
-- 이메일 시퀀스 추적
email_sequences (
  id, order_id, sequence_type, scheduled_at, sent_at, status
)

-- 재주문 예측
reorder_predictions (
  id, customer_email, product_id, avg_interval_days,
  last_order_date, next_predicted_date, reminder_sent
)
```

## 구현 순서
1. 원클릭 재주문 (즉시 효과, 프론트엔드만)
2. 장바구니 이탈 배너 (프론트엔드만)
3. 최근 본 상품 (프론트엔드만)
4. 주문 완료 관련 상품 추천 (프론트엔드만)
5. 자주 구매 제품 섹션 (프론트엔드 + API)
6. 이메일 시퀀스 (백엔드)
7. 재주문 주기 예측 (백엔드)
8. 프로덕션 빌드 + 리뷰
